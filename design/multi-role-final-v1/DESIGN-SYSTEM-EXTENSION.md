# 校园节奏多角色界面规范扩展

本规范承接 `design/student-final-v1/DESIGN-SYSTEM.md`，用于咨询师端和后台管理端。学生端既有视觉规则继续作为主规范，本文件只描述角色差异。

## 统一基础

- 页面底色：`#F7FAF9`
- 主文字：`#15262B`
- 次级文字：`#66787C`
- 品牌色：`#167A6E`
- 品牌浅色：`#EAF5F3`
- 风险色：`#E85A47`
- 分隔线：`#D9E2E0`
- 圆角：控件和内容容器统一不超过 `8px`
- 阴影：常规页面不使用；只允许抽屉等浮层使用克制阴影
- 信息结构：优先使用标题、分隔线、表格和列表，不堆叠装饰卡片

## 咨询师端

- 工作台采用移动端操作型布局，首屏先呈现待处理数量和状态筛选。
- 预约记录使用线性列表，当前待处理记录以左侧品牌色标识。
- 学生信息标注“仅用于本次咨询”，咨询记录页面保留敏感信息提示。
- 风险与未到访操作使用珊瑚色，普通状态使用青绿色。
- “填写咨询记录”是预约详情的主操作；“标记未到访”保持次级危险操作。

## 后台管理端

- 桌面端采用固定侧栏和主工作区，侧栏宽度 `248px`。
- 指标使用连续指标带，不拆成悬浮卡片。
- 表格保留稳定列宽，并在窄屏下只在表格容器内部横向滚动。
- 390px 窄屏下侧栏改为横向导航，表单改为单列，页面本体不得横向溢出。
- 焦点态统一为 `#167A6E` 的 2px 描边，保留键盘可访问性。

## 资产规则

- 使用真实位图资产 `breathing-lines.png` 作为克制的品牌辅助图案。
- 不使用 CSS 渐变纹理、CSS 线条插画、装饰圆球、Emoji 或文本符号模拟图标。
- 管理端以信息密度和扫描效率为优先，不增加与任务无关的插画。

- 后台管理端旧版：`design/source-archive/admin-ui-before-final-v1`
