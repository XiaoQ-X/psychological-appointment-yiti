# 学校心理预约小程序学生端设计系统

版本：v1.0

定稿日期：2026-07-12

视觉基准：`02-appointment-home.png`

适用范围：微信原生小程序学生端

## 1. 设计原则

- 核心任务优先：预约、查看预约和紧急求助必须在最短路径内完成。
- 青年校园感：清爽、有节奏、可信赖，不做娱乐化、营销化或医院化表达。
- 原生可预期：表单、弹层、按钮、加载与反馈遵循微信小程序交互习惯。
- 信息克制：每个页面最多一个主操作，列表使用分隔线而不是堆叠卡片。
- 安全明确：紧急求助固定使用珊瑚红语义，不与普通错误或取消操作混用。
- 隐私优先：不展示咨询师评分、预约热度、其他学生信息或公开评价。

## 2. 视觉令牌

### 2.1 颜色

| 令牌 | 色值 | 用途 |
| --- | --- | --- |
| `page` | `#F7FAF9` | 页面背景 |
| `surface` | `#FFFFFF` | 表单、列表和弹层表面 |
| `ink` | `#15262B` | 标题、重要数据 |
| `text` | `#3F5157` | 正文 |
| `muted` | `#748388` | 辅助说明、时间和占位文字 |
| `teal` | `#167A6E` | 主操作、当前状态、品牌识别 |
| `teal-pressed` | `#11655C` | 主操作按压态 |
| `teal-soft` | `#EAF5F3` | 轮播、确认状态、浅背景 |
| `sky` | `#3478E5` | 可约时段、信息型操作 |
| `sky-soft` | `#EEF5FF` | 时段选中弱背景 |
| `moss` | `#5C8A72` | 校内认证、成功辅助语义 |
| `coral` | `#E85A47` | 紧急求助、重要安全提醒 |
| `coral-soft` | `#FFF1EE` | 紧急入口和风险提示弱背景 |
| `amber` | `#B7791F` | 风险审核、注意事项 |
| `danger` | `#C9473D` | 取消、失败、不可逆操作 |
| `line` | `#D9E2E0` | 分隔线和输入框边界 |
| `disabled` | `#C8D1CF` | 禁用文字和按钮 |

禁止大面积渐变、紫色主色、单一蓝色页面和高饱和背景。一个页面最多使用一个主色与一个强调色。

### 2.2 字体

- 使用微信系统无衬线字体，不引入额外字体文件。
- 页面标题：22px / 30px，`600`。
- 分组标题：18px / 26px，`600`。
- 列表主信息：16px / 24px，`600`。
- 正文：15px / 23px，`400`。
- 辅助文字：13px / 20px，`400`。
- 状态与标签：12px / 18px，`500`。
- 数字时间使用等宽数字特性；字距固定为 `0`。

### 2.3 尺寸与间距

- 设计视口：390 x 844。
- 页面左右安全边距：16px；主要内容最大宽度：358px。
- 间距基线：8px；允许使用 4、8、12、16、20、24、32px。
- 控件最小触控高度：44px；主按钮高度：48px。
- 输入框高度：48px；多行输入最小高度：120px。
- 圆角：输入框和按钮 8px，标签 4px，头像使用圆形。
- 阴影仅用于底部固定操作栏和弹层；普通列表不使用阴影。

## 3. 图像与图案

- 咨询师头像使用统一光线、纯净中性背景的真实职业肖像，1:1 裁切。
- 顶部轮播禁止校园实景照片，采用文字主导的平面信息轮播。
- 轮播只允许低对比抽象路径、几何节奏或关怀符号，不使用叶片、渐变、光斑和装饰圆球。
- 页面图案不承载必要信息，不遮挡文字，首屏最多一处。
- 图标优先使用微信原生图标或一致的线性图标库，不使用 emoji。

## 4. 核心组件

### 4.1 顶部轮播

- 高度：112-128px；左右边距：16px；圆角：8px。
- 自动切换：4 秒；支持手势滑动和循环播放；用户操作后暂停 8 秒。
- 一次只显示一个主题，最多两行标题说明和一个文字操作。
- 底部显示 3 个分页圆点，当前圆点使用 `teal`。
- 推荐主题：考试周压力支持、首次咨询准备、预约与取消规则。

### 4.2 下一次预约

- 使用浅色横向信息带，不做独立大卡片。
- 必须同时显示日期、时段、咨询师、地点和状态。
- 整行可点击进入预约详情；状态不作为单独按钮。

### 4.3 日期选择器

- 一周 7 个等宽单元，固定高度，不因数字变化导致布局位移。
- 显示星期、月/日、可约数量；今天增加“今天”标记。
- 选中态使用 `teal` 实底；可用普通态使用白底；无时段使用弱灰。
- 选择日期后直接刷新列表，不增加冗余确认按钮。

### 4.4 咨询师列表行

- 头像 64px；姓名、校内认证和职称在首行。
- 擅长方向最多 3 个标签；地点和可约总量为辅助信息。
- 每行最多展示 2 个最近可约时段，其余通过“全部时段”查看。
- 不展示评分、销量、热度或公开评价。

### 4.5 状态标签

- 已确认、已完成：`teal-soft` + `teal`。
- 风险审核：浅琥珀底 + `amber`。
- 已取消：中性灰底 + `muted`。
- 失败或不可逆操作：`coral-soft` + `danger`。

### 4.6 表单

- 字段按任务分组，标签放在输入框上方。
- 必填字段显示文字“必填”，不只依赖红色星号。
- 校验错误紧邻字段显示；提交按钮只在信息完整后启用。
- 知情同意使用独立行入口，展示当前版本和阅读状态。

### 4.7 底部操作栏

- 固定在安全区上方，白底加顶部细线或极轻阴影。
- 每页最多一个实心主按钮；次要操作使用文字或描边。
- 危险操作不与主操作并排使用同等强调。

### 4.8 取消预约弹层

- 使用底部弹层，标题明确说明不可逆影响。
- 显示预约摘要、取消截止时间和单选原因。
- 未选择原因前禁用确认取消；主退出操作为“暂不取消”。

## 5. 页面与状态

| 编号 | 页面/状态 | 路由或归属 | 主要操作 |
| --- | --- | --- | --- |
| 01 | 学生登录 | `pages/login/login` | 登录 |
| 02 | 预约首页 | `pages/student/counselors/index` | 选择日期或咨询师 |
| 03 | 咨询师详情与时段 | `pages/student/counselor-detail/index` | 填写预约信息 |
| 04 | 预约信息表单 | `pages/student/booking-form/index` | 提交预约 |
| 05 | 知情同意 | `pages/student/consent/index` | 同意并返回预约 |
| 06 | 预约结果 | `pages/student/booking-result/index` | 查看预约详情 |
| 07 | 我的预约 | `pages/student/appointments/index` | 查看详情 |
| 08 | 预约详情 | `pages/student/appointment-detail/index` | 管理当前预约 |
| 09 | 取消预约弹层 | 预约详情状态 | 确认取消 |
| 10 | 已取消详情 | 预约详情状态 | 再次预约 |
| 11 | 紧急求助 | `pages/student/emergency/index` | 拨打求助电话 |

### 通用状态

- 加载：使用局部骨架屏，保留稳定高度，禁止整页白屏。
- 空数据：说明原因并提供一个合理行动，例如“查看其他日期”。
- 请求失败：保留已知内容，提供“重新加载”，不要清空整个页面。
- 提交中：按钮显示进度并阻止重复提交。
- 时段失效：明确提示“该时段刚被预约”，并刷新可用时段。
- 高风险：不展示诊断结论；提示学校将按流程联系并提供紧急求助入口。

## 6. 微信原生实现约束

- 后续直接在现有代码基础上修改，保留路由、接口字段、鉴权和预约并发逻辑。
- 先新增主题令牌和可复用组件，再逐页替换 WXML/WXSS，避免一次性重写。
- 原页面在全量验收前保留归档，不删除历史截图和设计证据。
- 轮播使用原生 `swiper`；日期和时段使用稳定网格尺寸。
- 所有按钮必须具备 `loading`、`disabled` 和按压反馈。
- 每页在微信开发者工具中以 390 x 844 截图，与对应高保真稿逐项比较。

## 7. 风格冻结规则

完成全页面设计并确认后，颜色、圆角、字体层级、间距体系、轮播结构和状态语义进入冻结状态。后续仅因业务变化、无障碍问题或真实设备兼容问题调整，并记录变更原因。
