# 学生端视觉与体验审核

审核日期：2026-07-11

审核范围：学生登录至预约取消、紧急求助的完整流程

证据来源：本目录 13 张微信开发者工具真实预览截图

## 总体结论

当前版本已经具备稳定的视觉基线和完整业务路径，但还不适合作为最终视觉稿直接交付。综合评级为 **B-：流程可用，需完成一轮真实数据、可读性和去模板化收口**。

优点是暖白、鼠尾草绿与珊瑚红的语义基本统一，预约、取消和紧急求助的状态区分清楚，按钮位置稳定，页面没有严重的卡片堆叠。主要问题不是“缺少装饰”，而是测试数据、英文眉题、过小文字和重复装饰让页面显得像设计模板；部分截图本身也不满足视觉验收条件。

## 验收阻断项 P0

1. **重新拍摄干净截图。** 多张截图带紫色鼠标光圈；`07-booking-result-devtools.png` 被桌面通知遮挡；登录截图仍显示历史控制台错误；部分截图显示开发工具历史警告。最终验收应使用无鼠标遮挡、无通知、控制台清空后的手机预览区域截图。
2. **替换全部测试数据。** `Integration Counselor`、`Counselor 20260704120708`、带编号的校区和房间明显是联调数据，会直接造成“不真实”和“AI 生成”的观感，也触发截断和拥挤。应使用 3-5 组真实长度的中文脱敏数据完成终稿复拍。
3. **补齐紧急求助配置。** 紧急求助页仍出现“待配置”。发布前必须由学校确认校内心理中心、值班电话、开放时间和线下地址；未配置的入口应隐藏或明确说明当前可用替代渠道，不能把占位状态直接展示给学生。
4. **结果页截图不合格。** 当前结果页核心预约卡被通知遮挡，无法确认字段完整性、长文本处理和实际视觉层级，必须单独复拍。

## 高优先级修改 P1

1. **减少英文眉题。** `MY APPOINTMENTS`、`APPOINTMENT DETAIL`、`CANCEL APPOINTMENT`、`INFORMED CONSENT`、`EMERGENCY SUPPORT`、`APPOINTMENT` 出现频率过高，形成统一模板套页的感觉。建议全部改为中文，或只在品牌首页保留一次英文辅助标识。
2. **提高正文可读性。** 多处辅助文字使用 `19-22rpx`，在 375px 设备上约为 9.5-11px，截图中已经明显偏小。正文建议不低于 `26rpx`，辅助文字不低于 `22-24rpx`，按钮文字使用 `28-30rpx`；浅绿色和浅灰色文字同时提高对比度。
3. **咨询师列表突出“可预约”。** 当前无时段咨询师排在有时段咨询师之前，学生需要额外判断。建议默认按最近可约时间排序，无可约时段项置底，并使用明确的“暂不可约”状态而不是只写“0 个时段”。
4. **增加可信身份信息。** 咨询师列表和详情缺少正式头像或学校认可的身份视觉，只使用字母圆形占位，容易显得像生成式模板。应接入学校提供的正式头像；没有头像时使用统一的官方默认头像资产，并显示职称、资质、校区和“校内认证”信息。
5. **压缩首页竞争入口。** “下次预约”和大面积“选择咨询师”横幅都在争夺首屏注意力。已有预约时，首页首要任务应是查看/管理本次预约，选择新咨询师降为次级入口；没有预约时再突出“开始预约”。
6. **重构知情同意阅读节奏。** 页面信息完整，但正文过小、段落密集、序号和标题间距接近。建议提高正文行高和字号，以六个明确章节分组，并增加“已阅读至底部”的进度反馈；同意按钮保持固定，但不能遮挡最后一段及勾选项。
7. **结果页减少空泛文案。** “预约已确认”之后应优先显示咨询师、日期、时间、地点、预约号和取消截止时间。鼓励性文案控制在一句，避免结果页出现大段泛化提示。
8. **列表记录增加扫描信息。** “我的预约”单行过于紧凑且下方大面积空白。记录应稳定展示日期、时间、咨询师、校区/地点、状态和下一动作；不必扩大成装饰性大卡片，但需提高行高与字段辨识度。

## 细节优化 P2

1. 呼吸线图案在多数页面重复出现，辨识度开始下降。建议只保留在登录、预约结果和紧急求助等关键页面，其余页面用留白和分隔线建立层级。
2. “欢迎回来”“你好，同学”“你已经在前进了”等文案偏通用。改为更具体的任务文案，例如“选择咨询师和可预约时间”“查看或调整你的预约”。
3. 登录页表单是典型白色卡片加输入框结构，品牌记忆较弱。可以加入学校心理中心正式名称、校徽/中心标识或经过授权的校园心理中心实景/插画资产，但不应增加无关渐变和装饰图形。
4. 日期、房间、校区等长文本目前依靠省略号，正式数据仍应做两行布局或独立地点行，避免关键信息被截断。
5. 预约详情中的预约号视觉权重过低，可以提供点击复制，但不应与页面标题争夺注意力。
6. 取消弹层结构清楚，建议补充最晚取消时间或规则影响，让学生在确认前理解后果；当前两级确认和固定操作栏可以保留。

## 无障碍风险

1. 小字号和低对比度是当前最明显的可感知风险，尤其是浅灰说明、绿色眉题、标签和房间信息。
2. 筛选标签、文字链接和部分小按钮的可点击区域从截图看可能低于 44×44px，需要在代码与真机中测量。
3. 选中状态不能只依赖绿色边框或颜色，应同时保留勾选图标、选中提示或状态文字。
4. 截图无法验证微信读屏标签、焦点顺序、动态字号、键盘弹起后的表单位置和错误提示播报，这些需要真机专项测试。

## 推荐实施顺序

1. 换真实中文脱敏数据并补齐紧急求助配置。
2. 统一中文字级、颜色对比度和最小触控尺寸。
3. 移除高频英文眉题和通用鼓励文案。
4. 调整咨询师排序、首页信息优先级和“我的预约”列表信息密度。
5. 接入正式头像/官方默认头像资产并补足身份信息。
6. 重新走完整流程，保存无鼠标、无通知、无控制台错误的干净截图。
7. 再用同尺寸原型图与实现截图做并排或叠图比较，才能给出像素级一比一结论。

## 审核边界

本审核确认了截图中可见的视觉层级、文案、布局、状态表达和部分交互反馈。仅凭截图不能证明完整无障碍合规、真机性能、弱网表现、键盘避让、屏幕阅读器支持或所有设备宽度下的响应式结果。
